# 🎆 赛博烟花启动页 - 完整设计方案

> **项目代号**：Ember Glow Fireworks Landing  
> **版本**：v1.0  
> **日期**：2025年  
> **优先级**：高（项目第一印象）

---

## 📋 需求回顾

在用户进入初始页（首页）**之前**，添加一个震撼的启动页：

1. **背景**：深夜晴空，极简黑色，星辰点缀
2. **烟花效果**：代码生成的赛博烟花，持续 15-20s，随机多样
3. **标题动画**：依次缓出三个元素，总耗时 5-10s
   - 《烟火漫游》（中文标题）- 上方
   - 《Ember Glow》（英文标题）- 中上方
   - 《进入系统》（按钮）- 下方
4. **布局**：所有元素水平居中，纵向分布
5. **交互**：按钮点击跳转到初始页
6. **集成目标**：最小化对前端项目的侵入，核心代码易于提取

## ✨ 核心方案设计

### 1️⃣ 可实现的烟花种类

#### **方案：多种粒子系统组合**

| 烟花类型 | 实现原理 | 视觉效果 | 代码复杂度 |
|---------|--------|--------|---------|
| **绽放型** | 从中心点向四周发射粒子，逐帧衰减 | 经典烟花，绚烂 | ⭐⭐ |
| **喷泉型** | 从下方连续喷射，模拟鞭炮喷泉 | 竖直律动感 | ⭐⭐ |
| **流星型** | 长条粒子划过，带尾迹效果 | 运动感强 | ⭐⭐⭐ |
| **环形扩展** | 粒子沿圆形轨迹扩展，逐圈变大 | 现代感，极简 | ⭐⭐ |
| **链式爆炸** | 多个爆炸点序列触发 | 层次感丰富 | ⭐⭐⭐ |
| **旋转花火** | 粒子沿螺旋轨迹运动 | 酷炫、科技感 | ⭐⭐⭐ |

**建议方案**：
- 基础层：**绽放型** + **环形扩展**（易于实现，视觉效果好）
- 进阶层：根据随机数种子，组合 **喷泉型** + **链式爆炸** + **旋转花火**

#### **随机种子机制**

```
每次页面加载 → 生成或读取种子 (可来自时间戳)
  ↓
根据种子生成：
  - 烟花数量 (8-12个)
  - 各烟花类型 (从6种中选择)
  - 爆炸位置 (网格随机)
  - 爆炸时机 (间隔随机)
  - 颜色方案 (主题色系随机)
  - 粒子数量 (30-100)
  
结果：每次启动界面都不同，但可复现
```

### 2️⃣ 背景图设置方案

#### **方案 A：纯代码渐变（推荐 ✅）**

```css
/* 极简夜空背景 */
background: radial-gradient(
  ellipse at center,
  #0a0e27 0%,    /* 深蓝黑 */
  #0f1419 50%,
  #000000 100%   /* 纯黑 */
);

/* 可选：加入星辰点缀 */
/* 使用 ::before 伪元素或 Canvas 绘制星点 */
```

**优点**：
- 无图片加载，极快响应
- 易于主题定制
- 兼容性强

**缺点**：
- 星辰需要额外代码实现

#### **方案 B：外部图片**

若队友有设计师制作的高质量背景图，可引入：
- 路径：`/public/assets/fireworks-bg.jpg` 或 `webp`
- 加载策略：使用 CSS 背景图 + `background-attachment: fixed`

### 3️⃣ 标题样式与动画

#### **中文标题：《烟火漫游》**

```css
font-family: "思源宋体", "SimSun", serif;  /* 古典文人感 */
font-size: 3.5rem;  /* 大约 56px */
font-weight: bold;
letter-spacing: 0.3em;  /* 文字间距，营造仪式感 */
color: #ffffff;  /* 纯白或 */
text-shadow: 
  0 0 20px rgba(255, 255, 100, 0.6),    /* 金色光晕 */
  0 0 40px rgba(255, 150, 50, 0.4);     /* 火焰橙晕 */

animation: fadeInSlideDown 2s ease-out;  /* 从上方淡入滑下 */
```

#### **英文标题：《Ember Glow》**

```css
font-family: "Playfair Display", serif;  /* 优雅衬线体 */
font-size: 2.5rem;  /* 大约 40px */
letter-spacing: 0.15em;
color: rgba(255, 255, 255, 0.85);  /* 略微透明 */
text-shadow: 
  0 0 15px rgba(100, 200, 255, 0.5);  /* 冷蓝光 */

animation: fadeInSlideDown 2.5s ease-out 0.8s both;  /* 延迟 0.8s 出现 */
```

#### **按钮：《进入系统》**

```css
/* 父容器 */
position: absolute;
bottom: 60px;
left: 50%;
transform: translateX(-50%);

/* 按钮样式 */
background: linear-gradient(45deg, #ff6b00, #ff9500);  /* 渐变橙 */
border: 2px solid rgba(255, 255, 255, 0.3);
color: #ffffff;
padding: 12px 40px;
font-size: 1.2rem;
border-radius: 50px;
cursor: pointer;
box-shadow: 
  0 0 20px rgba(255, 107, 0, 0.5),
  inset 0 1px 0 rgba(255, 255, 255, 0.3);

animation: fadeInScale 2s ease-out 1.5s both;  /* 延迟 1.5s 出现 */

/* Hover 效果 */
&:hover {
  box-shadow: 
    0 0 30px rgba(255, 107, 0, 0.8),
    0 8px 20px rgba(255, 107, 0, 0.4),
    inset 0 1px 0 rgba(255, 255, 255, 0.5);
  transform: translateX(-50%) scale(1.05);
}

/* 点击动画 */
&:active {
  transform: translateX(-50%) scale(0.95);
}
```

#### **关键动画定义**

```css
@keyframes fadeInSlideDown {
  0% {
    opacity: 0;
    transform: translateY(-40px);
  }
  100% {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes fadeInScale {
  0% {
    opacity: 0;
    transform: translateX(-50%) scale(0.8);
  }
  100% {
    opacity: 1;
    transform: translateX(-50%) scale(1);
  }
}

@keyframes pulse {
  0%, 100% {
    box-shadow: 0 0 20px rgba(255, 107, 0, 0.5);
  }
  50% {
    box-shadow: 0 0 30px rgba(255, 107, 0, 0.8);
  }
}
```

### 4️⃣ 动画时间规划

```
时间轴 (总长约 20-22s)
├─ 0s:      页面加载，烟花开始发射
├─ 0-18s:   烟花持续燃放（多个波次）
├─ 3-8s:    标题《烟火漫游》淡入滑下
├─ 3.8-9s:  标题《Ember Glow》淡入滑下
├─ 5-11s:   按钮《进入系统》淡入缩放出现
├─ 18-20s:  烟花逐渐淡出
└─ 20s:     用户可点击按钮（或自动跳转）
```

**时长选择理由**：
- 烟花 15-20s：足以展示多种花型，制造沉浸感
- 标题 5-10s 总出现时长：不过长，保持节奏感
- 存在重叠：创造"文字在烟花中出现"的视觉层次

## 🎨 核心代码架构建议

### **文件结构**

```
src/
├── components/
│   └── FireworksLanding/
│       ├── FireworksLanding.tsx       (主容器)
│       ├── FireworksCanvas.tsx        (Canvas 烟花渲染)
│       ├── TitleAnimation.tsx         (标题动画)
│       ├── EnterButton.tsx            (按钮)
│       └── styles.module.css          (所有动画 CSS)
├── utils/
│   └── fireworksEngine.ts             (烟花生成逻辑)
└── pages/
    └── Landing.tsx                    (整合页)
```

### **核心模块职责**

| 模块 | 职责 | 核心依赖 |
|------|------|--------|
| **FireworksCanvas** | 使用 Canvas 2D 或 Three.js 绘制烟花粒子 | pixi.js 或 Three.js (可选) |
| **fireworksEngine** | 根据种子生成烟花配置，管理粒子状态 | 无 |
| **TitleAnimation** | React 组件，渲染标题文字，CSS 驱动动画 | 无 |
| **EnterButton** | 按钮容器，路由跳转逻辑 | React Router 或 Next.js |

## 📦 集成到前端项目的策略

### **步骤 1：开发验证（你的电脑）**

1. 创建独立 React 项目：`npx create-react-app fireworks-landing`
2. 实现完整的启动页
3. 本地测试所有浏览器兼容性、性能

### **步骤 2：代码提取**

从项目中导出：
- `FireworksCanvas.tsx` + `fireworksEngine.ts` + `styles.module.css`
- 无其他依赖（纯 Canvas）

### **步骤 3：集成到队友项目**

队友在现有项目中：

```tsx
// src/pages/index.tsx (初始路由)
import { FireworksLanding } from '@/components/FireworksLanding'

export default function IndexPage() {
  return <FireworksLanding />
}
```

**关键点**：
- ✅ 零额外依赖（除非选用 Three.js）
- ✅ 纯 CSS 动画，无第三方库
- ✅ 可独立开发、独立测试、快速集成

## 🔧 技术选型推荐

### **烟花渲染方案对比**

| 方案 | 工具 | 性能 | 开发时间 | 代码复杂度 | 易集成性 |
|------|------|------|--------|----------|---------|
| **Canvas 2D** | 原生 | ⭐⭐⭐⭐ | 2-3h | 中 | ⭐⭐⭐⭐⭐ |
| **Three.js** | 3D库 | ⭐⭐⭐ | 4-5h | 高 | ⭐⭐⭐ |
| **PixiJS** | 2D库 | ⭐⭐⭐⭐⭐ | 2h | 低 | ⭐⭐⭐⭐ |
| **WebGL** | 原生 | ⭐⭐⭐⭐⭐ | 6h+ | 很高 | ⭐⭐ |

**我的建议**：**Canvas 2D** 最优
- 原生支持，零依赖
- 足以实现6种烟花效果
- 代码可控，易于定制
- 集成最简洁
